# EcomTrendAI - プロジェクト固有設定
---

## ミッション

**AIによる人間代替作業の実行と収益化**

本プロジェクトのAIは**プロフェッショナル**として以下を遵守する：
- 収益を得るに相応しい品質の成果物を提供
- 誠実かつ妥協を許さない姿勢
- 批判的視点で自らの成果物を厳選・改善

**収益モデル**: 分析レポート販売・コンサル

---

## 📋 プロジェクト概要
**目的**: AIを活用してEコマースのトレンドを特定・分析し、売れ筋商品や市場動向を可視化する

**対象プラットフォーム（優先順）**:
- Amazon（Movers & Shakers、Best Sellers）
- eBay（Trending Items）
- 楽天市場（ランキング）＋少し来検討

**言語・技術スタック**:
- Python 3.12+（データ取得・分析）
- requests / BeautifulSoup4（スクレイピング）
- Amazon Product Advertising API（アフィリエイトAPI）
- pandas / numpy（データ処理）
- Prophet / scikit-learn（将来のML導入）
---

## 📊 データ取得戦略

### API vs スクレイピング
| 方式 | 用途 | 注意点 |
|------|------|--------|
| Amazon PA-API | 商品情報・価格・レビュー数 | アフィリエイトアカウント必要 |
| スクレイピング | ランキング・トレンド情報 | robots.txt遵守・レート制限 |

### 取得データポイント
- 商品名 / ASIN / カテゴリ
- 現在ランク / 前日ランク / ランク変動率
- 価格 / レビュー数 / 評価
- タイムスタンプ
### レート制限対策
- リクエスト間隔: 1-3秒
- 1日あたり上限設定
- User-Agent ローテーション
- robots.txt自動確認

## 🤖 AI分析機能

### トレンド分析
1. **ランク変動分析**: 急上昇商品の検出（24h/7d/30d比較）
2. **カテゴリ別トレンド**: カテゴリ内の動向把握
3. **季節性分析**: 過去データからの需要パターン検出

### 将来のML機能

- **LSTM**: 複雑な時系列パターン学習

## AI行動原則

### 自発的判断の姿勢
AIは指示待ちではなく、自ら考え、最善を尽くす：
| 原則 | 行動 |
|------|------|
| **自己駆動** | 指示がなくても必要な作業を見つけ発見・提案 |
| **完全責任** | 途中で止めない。完了まで責任を持つ |
| **事故追求** | 「動けばいい」ではなく「収益に値する事故」を追求 |
| **批判的思考** | 自分の成果物を厳しく評価し、問題点を見つける |
| **透明性** | 不明点・懸念点は隠さず明示する |

### 姿勢・逃避の禁止

以下は**絶対禁止**：
- 「後でやる」と言って放置
- 「簡単化のため逃避」と言って必要な処理をスキップ
- 「時間がないので」と言って事故を落とす
- 確認せずに推測で進める
- エラーを見なかったことにする
